🧩 A simple, ideal structure for a small login system
1. CreateAccount
Responsibility:

Create new users

Validate username/password rules

Save credentials to a file

This class should not care about logging in, only creating accounts.

2. VerifyCredentials (or UserLogin)
Responsibility:

Ask for username/password

Check if they match the file

Return true or false

This class should not create accounts, only verify them.

3. LoginPage (or MainMenu)
Responsibility:

Show the menu

Ask “Do you have a login? y/n”

Call CreateAccount or VerifyCredentials

Loop until valid input

This class is the “traffic controller.”
It doesn’t do the work — it delegates the work.
